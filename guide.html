Unity Developer Guide: From 3D Model to Web Browser

Step 1: Asset Acquisition & Modeling Unity/Blender

You can create your own models or find free assets. The goal is a "Magical" object that is visually appealing but computationally light.

  • Finding Assets: Use Sketchfab, Poly Haven, or Unity Asset Store (Filter by Low Poly).
  • Modeling: If modeling in Blender or Unity, focus on a clean silhouette.
  • Optimization:
    • Keep the polygon count low (Target: under 20k tris per object).
    • Use Texture Baking to avoid complex real-time lighting calculations.
    • Limit the number of materials per object.

Step 2: Preparing for WebAR Optimization

Web browsers cannot handle high-end PC game assets. You must optimize for mobile hardware.

⚠️ IMPORTANT: WebAR will crash or lag if the model is too heavy.
  • Textures: Use power-of-two textures (e.g., 1024x1024 or 2048x2048).
  • Animations: If the character waves or floats, make sure the animation is baked into the skeleton/mesh.
  • Pivot Point: Ensure the model's pivot point is at the bottom center (0,0,0). This ensures it "sits" on the page instead of floating inside it.

Step 3: Conversion to GLB The Export

Unity uses .fbx or .prefab, but WebAR requires .glb (glTF Binary). This is the "JPEG of 3D."

Unity Tool: Install the "glTF-Export" package from GitHub or the Unity Package Manager.
  1. Go to File > Export glTF.
  2. Select .glb (not .gltf).
  3. Ensure "Include Animations" is checked if you have them.
  4. Test the resulting file using the Model-Viewer Editor to check for missing textures.

Step 4: The Hand-off Deployment

Once the .glb file is ready, it is passed to the web team for hosting.

Checklist for Hand-off:
[ ] File format is .glb
[ ] File size is < 10MB (Ideally < 5MB)
[ ] Pivot point is at the base
[ ] Texture are embedded in the GLB
